# AdSense 재심사 체크리스트

이 문서는 `one-billion-donation.com`의 재심사 준비용 내부 체크리스트입니다. Google은 별도의 최소 방문자 수를 공개된 자격 요건으로 제시하지 않습니다. 현재 우선순위는 트래픽 확대보다 **독창적인 콘텐츠, 명확한 운영 주체와 편집 기준, 사용하기 쉬운 사이트, 정책 준수, 사이트 소유권 확인**입니다.

공식 기준:

- [AdSense 참여 자격](https://support.google.com/adsense/answer/9724?hl=ko)
- [사이트를 AdSense에 연결하기](https://support.google.com/adsense/answer/7584263?hl=ko)
- [사이트 관리 및 검토 상태](https://support.google.com/adsense/answer/12131223?hl=ko)
- [사이트 소유권 확인](https://support.google.com/adsense/answer/91205?hl=ko)
- [고유 콘텐츠·투명성·탐색성 점검](https://support.google.com/adsense/answer/1348727?hl=ko)

## 1. 코드와 콘텐츠 배포

- [ ] 이번 콘텐츠 개선 사항을 검토하고 커밋한다.
- [ ] 원격 저장소에 푸시한 뒤 GitHub Pages 배포가 완료됐는지 확인한다.
- [ ] 한국어·영어 홈, 소개, 방법론, 개인정보처리방침, 면책 조항, 저작권 안내가 공개 URL에서 열린다.
- [ ] 대표 콘텐츠 10개에서 출처, 산정 방식, 가정과 한계가 실제 공개 페이지에 표시된다.
- [ ] 모바일과 데스크톱에서 메뉴, 본문, 표, 이미지가 겹치거나 잘리지 않는다.
- [ ] 깨진 링크와 누락된 이미지가 없다.

## 2. 콘텐츠 가치와 편집 신뢰성

- [ ] 홈에서 이 사이트가 직접 작성한 분석과 독자가 얻을 수 있는 가치가 바로 보인다.
- [ ] 과도한 수익 보장·절대적 우위·항상 성립한다는 표현이 남아 있지 않다.
- [ ] 시뮬레이션, 백테스트, 2차 데이터는 실적이나 공식 1차 데이터처럼 표현하지 않는다.
- [ ] `회사원·개인 투자자`라는 운영자 배경과 비전문가 한계가 소개 페이지에 공개되어 있다.
- [ ] AI·소프트웨어 보조 사용과 최종 검토 책임이 소개 페이지에 공개되어 있다.
- [ ] 보유 종목, 제휴, 후원 등 이해상충이 생기면 해당 글에 눈에 띄게 밝힌다.
- [ ] 짧거나 중복된 글을 수만 늘리기 위해 새로 게시하지 않는다.

## 3. 검색 노출과 크롤링

- [x] `https://one-billion-donation.com/robots.txt`가 열리고 전체 크롤링을 허용한다.
- [x] `https://one-billion-donation.com/sitemap.xml`이 열리고 새 페이지가 포함되어 있다.
- [ ] Google Search Console에 사이트맵을 다시 제출한다.
- [ ] 홈, 소개, 방법론과 대표 콘텐츠에 URL 검사를 실행하고 색인을 요청한다.
- [ ] `noindex`, 로그인 벽, 지역 차단, 과도한 리디렉션으로 Google 크롤러를 막지 않는다.
- [ ] 검색 결과에 이전 제목·설명이 남아 있어도 새 배포의 재크롤링 시간을 기다린다.

## 4. AdSense 사이트 연결

- [ ] `docs/public/ads.txt`의 `pub-6597757677689519`가 실제 AdSense 계정의 게시자 ID와 정확히 일치하는지 확인한다.
- [ ] AdSense의 **사이트** 화면에 `one-billion-donation.com`을 추가한다.
- [ ] AdSense가 제시하는 방법 중 하나로 소유권을 확인한다: AdSense 코드, `ads.txt`, 또는 메타 태그.
- [ ] 코드나 메타 태그를 선택했다면 AdSense 화면에서 받은 값을 그대로 사용한다. 예시·임의 ID를 넣지 않는다.
- [ ] 공개 URL에서 `google-adsense-account` 메타 태그와 `ads.txt`가 실제로 제공되는지 확인한다.
- [ ] AdSense의 사이트 상태가 소유권 확인 실패나 크롤러 접근 불가가 아닌지 확인한다.

현재 저장소에는 `ads.txt`와 `google-adsense-account` 확인용 메타 태그가 있습니다. Google Analytics는 방문자 분석 목적으로 로드하며 개인정보처리방침에 이를 공개했습니다. AdSense 광고 런타임은 개인정보 고지와 동의 관리가 준비될 때까지 로드하지 않습니다.

## 5. 개인정보와 광고 운영 준비

- [x] 현재 Google Analytics와 AdSense 연결 상태가 개인정보처리방침의 설명과 일치한다.
- [ ] 광고를 실제로 활성화하기 전 개인정보처리방침을 광고·쿠키의 실제 동작과 다시 일치하도록 갱신한다.
- [ ] 적용 지역과 방문자에 따라 Google이 요구하는 동의 관리 플랫폼(CMP) 또는 동의 선택 화면을 준비한다.
- [ ] 광고가 콘텐츠, 메뉴, 다운로드 버튼처럼 오인되지 않게 배치한다.
- [ ] 본문보다 광고가 더 두드러지거나 첫 화면을 과도하게 차지하지 않게 한다.
- [ ] 자동 광고를 켠 뒤 모바일 레이아웃과 Core Web Vitals를 다시 확인한다.

## 6. 재심사 요청

- [ ] 위 변경이 로컬이 아니라 공개 사이트에 모두 반영된 뒤 요청한다.
- [ ] 대표 페이지가 크롤링 가능한지 먼저 확인한다.
- [ ] AdSense 사이트 연결과 소유권 확인을 완료한 뒤 검토를 요청한다.
- [ ] 검토 중에는 URL 구조, 사이트 소유권 방법, 핵심 콘텐츠를 반복해서 크게 바꾸지 않는다.
- [ ] Google 안내상 검토는 보통 며칠이지만 경우에 따라 2~4주가 걸릴 수 있으므로, 상태가 진행 중인 동안 중복 요청하지 않는다.

## 재심사 뒤에도 계속할 일

- 새 글마다 작성 목적, 독자에게 주는 고유 가치, 근거, 방법, 가정, 한계를 점검한다.
- 월 1회 오래된 수치·외부 링크·검토일을 확인한다.
- 검색 유입이 적더라도 얇은 글을 양산하지 않고 기존 핵심 글을 실제 사례와 원자료로 보강한다.
- Search Console에서 색인 제외 사유와 검색어를 확인하되, 검색어 반복 삽입보다 독자의 질문에 완결된 답을 주는 것을 우선한다.
